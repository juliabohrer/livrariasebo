## Descrição do Projeto

### Visão Geral do Projeto

O Sebo Digital é uma plataforma de e-commerce focada em livros usados. Sua finalidade principal é criar um ponto de encontro digital para o mercado de segunda mão de livros, garantindo que títulos raros e esgotados possam ser facilmente encontrados e adquiridos.

O projeto trará o benefício de:

- Ampliar significativamente o público do sebo.
- Melhorar a organização do estoque.
- Proporcionar uma nova fonte de receita com baixo custo operacional.

As principais funcionalidades envolvem busca avançada por livros, gestão de catálogo (CRUD), carrinho de compras, checkout seguro e um painel administrativo completo.

O objetivo deste documento é fornecer uma visão geral do projeto Sebo Digital, descrevendo sua finalidade, os principais _stakeholders_ envolvidos, os requisitos principais, os diagramas de caso de uso, diagrama de classes e protótipos de possíveis telas para o sistema.

### 2. Stakeholders

A tabela a seguir identifica as principais partes interessadas envolvidas no projeto e descreve o papel de cada uma:

| Stakeholder                        | Descrição do Papel                                                                                                                               |
| :--------------------------------- | :----------------------------------------------------------------------------------------------------------------------------------------------- |
| **Dono do Sebo (Cliente)**         | Fornece o domínio de conhecimento sobre livros usados, define a estratégia de preços e é o decisor final sobre funcionalidades e prioridades.    |
| **Usuário Comum (Comprador)**      | Usuário final que busca, navega, compra livros e acompanha pedidos. Seu feedback é crucial para a usabilidade e o sucesso da plataforma.         |
| **Administrador do Sistema (ADM)** | Responsável por manter o catálogo (**CRUD**), gerenciar o estoque, processar pedidos e gerenciar contas de usuários na plataforma.               |
| **Equipe de Desenvolvimento**      | Responsável pela arquitetura, codificação, testes e implantação do sistema, garantindo que os requisitos sejam atendidos de acordo com o escopo. |
